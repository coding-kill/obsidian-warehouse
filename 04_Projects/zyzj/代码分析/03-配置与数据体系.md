# 类别三：配置与数据体系

## 3.1 配置加载机制

```
┌──────────────────────────────────────────────────────────────┐
│                    配置加载顺序流程图                           │
└──────────────────────────────────────────────────────────────┘

┌─────────────────┐
│   my.config     │  ← 只存"当前环境目录名"（dev/live）
└────────┬────────┘
         │ 读取环境目录名
         ▼
┌─────────────────────────┐
│   MyServerInfo.json      │  ← 本机默认world/server、日志、线程池等
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│   ServerInfo_global_*.json           │  ← 全局层服务 + 全局DB/Redis + 地图组
└────────┬────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────┐
│   ServerInfo_world_{id}_*.json          │  ← 某world下的服务、DB、Redis、容量
└────────┬────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────┐
│   根据启动参数确定"我是哪个服务实例"        │
│   (worldId/serverId/...)                 │
└─────────────────────────────────────────┘
```

### 核心配置字段

| 字段 | 含义 |
|------|------|
| server_id | 服务唯一标识 |
| world_id | 世界/区服ID |
| terrain_group_id | 地图分组（决定哪些game/war服承载哪些地图） |
| ip/port | 网络地址 |
| server_list | 该层所有服务实例列表 |

### 配置路径

```
TCPServer/Bin/config/
├── my.config                      → 环境目录切换
├── MyServerInfo.json              → 本机基础参数
├── ServerInfo_global_*.json        → 全局层配置
└── ServerInfo_world_{id}_*.json   → world层配置
```

## 3.4 发货/商品映射机制

### 渠道商品映射表结构

```json
{
  "PRODUCT_INDEX": 101020080,    // 游戏内商品主索引
  "PRODUCT_SUB_KEY": 1,          // 商品子键（与INDEX组成唯一键）
  "GOOGLE_ID": "10814",          // Google Play商店商品ID
  "ONE_STORE_ID": "10818",       // OneStore商店商品ID
  "IOS_ID": "10822",             // Apple商店商品ID
  "PC_ID": "0"                   // PC渠道商品ID（0表示未配置）
}
```

**映射关系**：`外部商店商品ID` → `(PRODUCT_INDEX, PRODUCT_SUB_KEY)` → 游戏内商品

**业务链路**：
```
外部支付回调 → BillingServer → 映射表反查 → ShopProductInfo → 发货
GM付费礼包 → itemNo(PC_ID) → ShopItemInfo → (GroupIdx, GroupKey) → 发货
```
