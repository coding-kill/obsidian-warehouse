# 类别六：运维与排障指南

## 6.1 启动期报错字典

### 1. DataManager Init failed!

| 项目 | 说明 |
|------|------|
| 日志 | `DataManager Init failed!` |
| 入口代码 | `GameServer.cpp:70-74` |
| 归属模块 | GameServer启动阶段 |
| **第一优先检查** | 最近改动的json是否有语法问题（逗号/引号/尾逗号） |

```cpp
if (!DATA_TABLE_MGR->Initialize())
{
    LOG_ERROR << "DataManager Init failed!";
    return false;
}
```

---

### 2. dataTable load fail!

| 项目 | 说明 |
|------|------|
| 日志 | `dataTable load fail!` |
| 入口代码 | `DataTableManager.cpp:107-113` |
| 归属模块 | 任意`read_xxxTable()`返回false |
| **第一优先检查** | 找最后一个"成功读表"之后的下一个表 |

```cpp
if (result)
    LOG_INFO << "dataTable init complete!";
else
    LOG_ERROR << "dataTable load fail!";
```

---

### 3. Item Manager load failed!

| 项目 | 说明 |
|------|------|
| 日志 | `Item Manager load failed!` |
| 入口代码 | `GameServer.cpp:315-318` |
| 归属模块 | ItemManager |
| **第一优先检查** | ItemBasicTable新增item重复ID/非法类型 |

```cpp
if (false == ITEM_MGR->loadData(DATA_TABLE_MGR->getItemTable()))
{
    LOG_ERROR << "Item Manager load failed!";
    return false;
}
```

---

### 4. dataTable - check relation 失败

| 项目 | 说明 |
|------|------|
| 日志 | `dataTable - check relation complete!` 不出现 |
| 入口代码 | `DataTableManager.cpp:420-427` |
| 归属模块 | `checkRelation_*` 系列函数 |
| **第一优先检查** | 跨表引用断链、等级/阶段不连续 |

### 10. 改了技能但战斗中异常

| 高概率根因 | 排查顺序 |
|-----------|---------|

| | 4. SkillAbilityValueTable有对应value |
| | 5. 当前stance确实挂了技能 |

## 6.3 改表防炸服作业单

### 按改动类型分类

#### A. 角色改动必查

| 涉及表 | 高危报错 |
|--------|---------|
| CharacterBasicTable | 新角色没配置导致创建失败 |
| CharacterPromotionTable | 职业/类型改了但promotion未同步 |
| CharacterInitialTable | stance初始化依赖 |
| StanceBasicTable | 当前stance无效 |

**固定回归**：
1. 新建角色 → 进游戏 → 切stance

---

#### B. 技能改动必查

| 涉及表 | 高危报错 |
|--------|---------|
| SkillBasicTable | skill存在但skillLevel缺行 |
| SkillLevelTable | ability id在SkillAbilityValue找不到 |
| StanceBasicTable | 配置了无效/锁定/不可见技能 |

**固定回归**：
1. 主动技能释放
2. 技能升级
3. 冷却时间

---

#### C. 卡牌改动必查

| 系统 | 涉及表 | 高危报错 |
|------|--------|---------|
| Pioneer | PioneerTable, PioneerRein*, Pro* | 强化阶段/概率不一致 |
| Collection | CollectionBasic*, Hero/Pet/Item/MonsterCollection* | 子表关系缺失 |
| Pet | PetBasic*, Incubation*, Synthetic*, Exchange* | 蛋物品不匹配、结果pet不存在 |

**固定回归**：
1. pioneer强化
2. collection注册/领奖
3. pet孵化/合成/兑换

## 6.5 快速检查脚本（人工版）

每次策划改完，按顺序执行：

```
□ 1. 是否打印 "dataTable init complete!"
□ 2. 是否打印 "dataTable - check relation complete!"
□ 3. 是否出现 "Item Manager load failed!"
□ 4. 登录一个老号 + 新建一个新号
□ 5. 执行：切stance -> 放技能 -> 卡牌核心操作
```
