# 类别七：快速上手指南

## 7.1 五天学习计划

### Day 1：跑通最小链路

**目标**：理解服务启动与注册关系

**步骤**：
1. 启动 `GlobalServer + LoginServer`
2. 启动 `World + DBAgent + Community`
3. 启动 `GameServer`
4. 观察日志确认服务注册关系（谁连谁）
5. 用客户端登录，确认进入游戏

**预期日志**：
```
[WorldServer] World Server Connected
[CommunityServer] Community Server Connected
[DBAgentServer] DBAgent Server Connected
[dataTable init complete!]
[Item Manager load success]
```

### Day 2：理解GameServer启动依赖

**目标**：理解配置-依赖-初始化链

**步骤**：
1. 阅读 `GameServer.cpp` 的 `startingService()`

**核心文件**：
```
GameServer.cpp:70-323  ← 启动主流程
Config/CConfig.*      ← 配置中枢
DataTableManager.cpp  ← 表加载中枢
```

### Day 3：理解Game-World通信

**目标**：掌握服务间消息收发模型

**步骤**：
1. 阅读 `WorldSrvManager` 理解Game-World通信
2. 查看 `sendConnectOKSign()` 发送哪些注册信息
3. 查看 `sNetRecv_UserConnect` 处理用户接入
4. 追踪一个具体包（如 `NtfyUserKick`）的完整流转

**核心文件**：
```
WorldSrvManager.*        ← Game-World通信中枢
WorldSrvConn.*           ← World连接管理
User_Handler_WorldSrv.cpp ← World下发包处理
```

---

### Day 4：理解Web服务入口

**目标**：理解ASP侧服务架构

**推荐**：`LoginServer`

**步骤**：
1. 阅读 `LoginServer/Program.cs` 理解端口绑定/鉴权
2. 查看 `ServerInfoManager` 如何加载配置
3. 理解 FlatBuffers Formatter 如何工作
4. 追踪一次登录请求的完整链路

**核心文件**：
```
LoginServer/Program.cs     ← 入口
Asp/CommonLib/BaseApp.cs  ← 全局静态上下文
Asp/CommonLib/ServerInfoManager.cs ← 配置中枢
```

---

### Day 5：跟一遍完整包流转

**目标**：串联所有知识点

**推荐链路**：登录后进服

**步骤**：
1. 客户端发送登录请求 → LoginServer
2. LoginServer 验证 → 返回token + 世界信息
3. 客户端连接 GameServer → WorldServer 注册
4. UserConnect → 创建User对象 → 进入usingList
5. 客户端发 CharacterSelect → WorkThread处理
6. 加载角色数据（DBAgent）→ 返回角色信息
7. 进入游戏 → TerrainManager 加载地图

## 7.3 扩展机制

### A. 新增服务实例（横向扩容）

```
步骤：
1. 在 ServerInfo_world_xxx.json 增加 server_list 项
   {
     "name": "GAME",
     "server_id": xxx,
     "ip": "xxx",
     "port": xxx,
     "terrain_group_id": 3
   }
2. 配置对应端口/ID/terrain_group
3. 由批处理或运维脚本拉起
```

### B. 新增业务协议

```
步骤：
1. 新增 FlatBuffers 消息定义（PID + schema）
2. 在对应 SessionManager 的 packetProcedure 注册处理
3. 在 SendFunc/Manager 中增加发送封装
```

### C. 新增数据表

```
步骤：
1. 加 datatable（JSON）和 SQL
2. 在 DataManager 侧加载
   bool read_xxxTable("xxxTable.json")
3. 在业务 manager 初始化时接入
```

### D. 新增Web API

```
步骤：
1. ASP对应服务新增 Controller/API
2. 用 ServerInfoManager 读取配置/目标世界
3. 复用 CommonLib 的 DB/Env/加密/证书工具
```

## 7.4 风险注意点

| 风险 | 说明 |
|------|------|
| 端口与server_id强绑定 | 改错会"能启动但互联失败" |
| my.config决定环境目录 | 很多"读不到配置"根源在此 |
| DB/Redis密码可能加密 | 不要只改json明文 |
| C++与ASP配置要双向一致 | 共享同一拓扑概念 |
| 多服务编排 | 单点调试需要mock或最小集群 |
