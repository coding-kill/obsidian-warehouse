# 类别一：架构与服务体系

## 1.1 项目整体架构

卓越之剑OPUS是一个**分层多进程MMO RPG集群**：

```
┌─────────────────────────────────────────────────────────────┐
│                        客户端                                │
└─────────────────────────────┬───────────────────────────────┘
                              │
        ┌─────────────────────┴─────────────────────┐
        │              ASP 服务集群 (.NET 8)         │
        │  LoginServer / GeAsp / ChatServer /       │
        │  AuctionServer / ArenaServer / Billing...  │
        └─────────────────────┬─────────────────────┘
                              │ HTTP/TCP
┌─────────────────────────────┴───────────────────────────────┐
│                      C++ 服务集群                             │
│  ┌─────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ Global   │  │  World   │  │Community │  │ DBAgent  │   │
│  │ Server   │  │  Server  │  │  Server  │  │  Server  │   │
│  └─────────┘  └──────────┘  └──────────┘  └──────────┘   │
│  ┌─────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │  Game    │  │ Instance │  │ Instance │  │   Log    │   │
│  │  Server  │  │  Pool    │  │  Server  │  │  Server  │   │
│  └─────────┘  └──────────┘  └──────────┘  └──────────┘   │
└──────────────────────────────────────────────────────────────┘
```

### C++ 服务集群职责矩阵

| 服务名 | 职责 | 关键代码路径 |
|--------|------|------------|
| GlobalServer | 全局控制面，跨world总入口 | - |
| WorldServer | world级协调中心（频道、跨服状态、用户注册） | WorldSrvManager.* |
| CommunityServer | 社区/关系域（组队、公会、社交协调） | CommunitySrvManager.* |
| GameServer | 核心玩法逻辑（角色、地图、战斗、任务、道具等） | GameServer.*, User.* |
| InstancePoolServer | 副本实例池调度 | - |
| InstanceServer | 具体副本执行节点 | - |
| DBAgentServer | C++服务访问数据库的中介层（多连接分片） | DBAgentSrvClient.* |
| LogServer | 日志聚合/持久化通道 | - |

### C# 服务集群职责矩阵

| 服务名 | 职责 | 关键代码路径 |
|--------|------|------------|
| LoginServer | 登录鉴权入口 | LoginServer/Program.cs |
| GeAsp | 综合业务API网关 | GeAsp/Program.cs |
| ChatServer | 世界公告聚合发送 | ChatServer |
| AuctionServer | 拍卖行HTTP接口 | AuctionServer |
| ArenaServer | 竞技场接口 | ArenaServer |
| WorldQueueServer | 登录排队 | WorldQueueServer |
| BillingServer | 支付/计费通道 | BillingServer |
| AuthenticationServer | 第三方认证校验 | AuthenticationServer |

## 1.2 启动编排顺序

```
┌────────────────────────────────────────────────────────────────┐
│                      启动顺序流程图                               │
└────────────────────────────────────────────────────────────────┘

[GlobalServer + LoginServer]  ──►  [World层服务集群]
                                        │
                                        ├─ LogServer
                                        ├─ DBAgentServer (多连接分片)
                                        ├─ WorldServer
                                        ├─ CommunityServer
                                        ├─ InstancePoolServer
                                        ├─ GeAsp
                                        ├─ ChatServer
                                        ├─ WorldQueueServer
                                        ├─ AuctionServer
                                        └─ ArenaServer (可选)
                                              │
                                              ▼
                                    [GameServer 层] ◄──► [InstanceServer]
                                              │
                                              ▼
                                       [WarServer 层]
```

**推荐启动顺序**：
1. 基础服务：DB/Redis/认证服务
2. World + Community + DBAgent + Log
3. Chat/Auction/Billing/Arena 等业务外围服务
4. **最后启动 Game/Instance 服**（Game启动后会立刻尝试连接上述服务）

## 1.3 技术栈总结

| 层级 | 技术选型 |
|------|---------|
| 游戏核心逻辑 | C++ (VS工程, IOCP模型) |
| Web/网关/业务接口 | C# .NET 8 (Kestrel) |
| 数据库 | SQL (account/game/world/guild/log分库) |
| 缓存 | Redis |
| 配置中心 | JSON配置文件 + my.config |
| 通信协议 | FlatBuffers |
