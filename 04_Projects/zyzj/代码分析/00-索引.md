## 关键结论一句话版

### 架构
- **分层多进程MMO集群**：C++游戏服 + C# Web服 + 分库DB + Redis

### 通信
- **IOCP + 工作线程池 + FlatBuffers**：网络线程收包投递，业务在工作线程执行
- **统一重连框架**：SrvConnectThread每100ms检查，断线自动重连并重新RegServer

### 配置
- **my.config → MyServerInfo → global → world**：四层加载，按启动参数确定服务身份

### 数据
- **依赖链驱动加载顺序**：SkillUpgrade → Ability → Value → Basic → Level → Stance
- **checkRelation()校验跨表引用完整性**

### 用户
- **Online → Wait → Return**：短离线保留内存，超时回收对象池
- **跨服单主写 + saveUser前置落地 + World协调迁移**

### 业务
- **GameServer include列表即系统全家桶**：角色/技能/卡牌/地图/社交/PVP/活动全覆盖

### 排障
- **日志关键字驱动**：DataManager Init failed / dataTable load fail / Item Manager load failed / not found curr Stance
