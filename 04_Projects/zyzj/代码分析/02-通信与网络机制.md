# 类别二：通信与网络机制

## 2.1 服务间通信（C++ TCP长连接）

```
┌──────────────────────────────────────────────────────────────┐
│                    IOCP + 工作线程池 模型                      │
└──────────────────────────────────────────────────────────────┘

                      ┌─────────────┐
                      │   IOCP      │  ← 网络层（收包）
                      │  端口监听   │
                      └──────┬──────┘
                             │
                    ┌────────▼────────┐
                    │    Session     │  ← 会话管理
                    │    包池        │
                    └──────┬─────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
     ┌────────▼─────┐ ┌──────▼──────┐ ┌─────▼──────┐
     │ WorkThread 1 │ │ WorkThread 2│ │ WorkThread N│  ← 工作线程池
     │              │ │             │ │             │
     └────────┬─────┘ └──────┬──────┘ └─────┬──────┘
              │              │              │
              └──────────────┼──────────────┘
                             │
                    ┌────────▼────────┐
                    │  PacketManager  │  ← 包对象池
                    └─────────────────┘
```

### 通信流程

**收包**：`IOCP` → `Session` → 投递`WorkThread` → `PacketProcedure` → 路由到具体Handler

**发包**：`业务逻辑` → 构建`FlatBuffers Builder` → `Session.send()`

### GameServer 启动后主动连接的服务

```
GameServer ─┬─► WorldServer
            ├─► CommunityServer
            ├─► DBAgentServer (多连接，按uid分片)
            ├─► LogServer
            ├─► ChatServer (HTTP聚合)
            ├─► AuctionServer (HTTP)
            ├─► BillingServer (HTTP)
            └─► ArenaServer (条件)
```

## 2.2 服务注册与重连机制

```
┌──────────────────────────────────────────────────────────────┐
│                    统一重连框架流程图                           │
└──────────────────────────────────────────────────────────────┘

    ┌─────────────┐      ┌─────────────────┐      ┌────────────┐
    │  服务启动    │ ───► │ connect()       │ ───► │ postConnect│
    └─────────────┘      └─────────────────┘      └─────┬──────┘
                                                        │
                              ┌─────────────────────────┘
                              ▼
                    ┌─────────────────┐
                    │ sendConnectOKSign│  ← 发送RegServer注册包
                    │ (本服信息注册)   │
                    └────────┬────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │ SRV_CONNECT_MGR │  ← 加入自动重连监控列表
                    │ .addSrv()       │
                    └────────┬────────┘
                             │
                             ▼
              ┌──────────────────────────────────┐
              │     SrvConnectThread             │
              │     (每100ms轮询所有服务)          │
              │     checkConnect()               │
              └──────────────┬───────────────────┘
                             │
              ┌──────────────▼──────────────┐
              │   条件: !m_bActive && !isAlive()│
              └──────────────┬──────────────┘
                             │ 是
              ┌──────────────▼──────────────┐
              │  重新 initialize()         │
              │  重新 connect()             │
              │  重新 postConnect()         │
              │  重新 sendConnectOKSign()   │
              └─────────────────────────────┘
```

### 不同服务的重连策略

| 服务类型 | 重连机制 | 说明 |
|---------|---------|------|
| World/Community/DBAgent/InstancePool | 统一重连框架 | SrvConnectThread每100ms检查 |
| LogServer | 独立重连线程 | 自有LogSrvConnectThread |
| AuctionServer | HTTP请求式 | 无TCP长连，不走RegServer |
| BillingServer | HTTP队列 | 无TCP长连 |
| ChatServer | HTTP聚合发送 | 无TCP RegServer |
| AuthenticationServer | 一次性HTTPS | 无重连机制 |

## 2.4 核心协议注册信息

### WorldServer 注册信息

**GameServer → WorldServer** 发送 `RegServer`，字段：
- serverId、NetServerInfo（worldId/serverId/ip/port/name/terrainIdxGroupId）
- terrainList（地形注册信息）

**WorldServer → GameServer** 接收并保存：
- `sNetRecv_RegServer` → 保存channelList到各terrain
- `sNetRecv_UserConnect` → 创建/租赁User、绑定Session
- `sNetRecv_NtfyUserKick` → 触发用户踢下线

### DBAgentServer 注册信息

**GameServer → DBAgent** 每个connector发送 `RegServer`：
- NetServerInfo（worldId/serverId/ip/port2/name/terrainIdxGroupId）

**分片路由**：`dbShardingUid % threadCount` 选择发送线程
